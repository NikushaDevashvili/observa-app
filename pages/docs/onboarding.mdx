# Onboarding

Complete onboarding guide to get you started with Observa.

## Account Setup

### 1. Sign Up

1. Visit [observa-app.vercel.app](https://observa-app.vercel.app)
2. Click **Sign Up**
3. Enter your email and create a password
4. Verify your email address

### 2. Complete Initial Setup

After signing up, you'll be taken through the onboarding flow:

1. **Account Creation**: Your account is automatically created
2. **API Key Generation**: Your API key is generated and displayed
3. **Project Setup**: Your default project is created
4. **Environment Selection**: Choose your environment (development/production)

### 3. Save Your Credentials

Make sure to save:
- **API Key**: Copy this immediately (you can view it later in Settings)
- **Tenant ID**: Your tenant identifier
- **Project ID**: Your project identifier

## First Integration

### Step 1: Install SDK

```bash
npm install observa-sdk
```

### Step 2: Initialize SDK

```javascript
import { init } from 'observa-sdk';

const observa = init({
  apiKey: 'your-api-key-from-signup',
  apiUrl: 'https://observa-api.vercel.app',
});
```

### Step 3: Track Your First Event

```javascript
await observa.track(
  {
    query: "Hello, Observa!",
    conversationId: "first-conversation",
    messageIndex: 1,
  },
  async () => {
    // Your LLM call
    return { response: "Hello! I'm ready to track your LLM calls." };
  }
);
```

### Step 4: Verify in Dashboard

1. Go to [Dashboard](https://observa-app.vercel.app/dashboard)
2. Navigate to **Traces**
3. You should see your first tracked event

## Best Practices

1. **Environment Variables**: Always use environment variables for API keys
2. **Error Handling**: Wrap tracking calls in try-catch blocks
3. **Session Management**: Use conversation IDs to group related messages
4. **Metadata**: Add relevant context to your tracking calls
5. **Testing**: Test in development before deploying to production
