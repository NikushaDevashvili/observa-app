# Issues & Alerts

Monitor and resolve issues in your LLM application.

## Accessing Issues

Navigate to **Issues** in the dashboard sidebar to view all detected problems.

## Issue Types

Observa tracks various issue types:

- **Errors**: Failed LLM calls
- **High Latency**: Slow responses
- **High Cost**: Expensive operations
- **Quality Issues**: Poor responses
- **Rate Limits**: API rate limit hits

## Issue List

The issues page shows:

- **Type**: Issue category
- **Severity**: Critical, warning, or info
- **Count**: Number of occurrences
- **First Seen**: When first detected
- **Last Seen**: Most recent occurrence
- **Status**: Open or resolved

## Issue Details

Click on an issue to see:

### Issue Information
- Type and description
- Severity level
- Status
- Affected traces

### Occurrences
- List of all occurrences
- Timeline view
- Related traces
- Context information

### Resolution
- Steps to resolve
- Related documentation
- Similar issues
- Resolution history

## Alerting

Set up alerts for:

- Error rate thresholds
- Latency spikes
- Cost anomalies
- Quality degradation

## Resolving Issues

### Common Issues

1. **High Error Rate**
   - Check API keys
   - Verify network connectivity
   - Review error messages

2. **High Latency**
   - Optimize prompts
   - Use faster models
   - Review infrastructure

3. **High Costs**
   - Use cheaper models
   - Optimize token usage
   - Implement caching

4. **Quality Issues**
   - Improve prompts
   - Add context
   - Fine-tune models

## Best Practices

1. **Monitor Regularly**: Check issues dashboard frequently
2. **Set Alerts**: Get notified of problems
3. **Investigate Root Causes**: Don't just fix symptoms
4. **Track Resolution**: Document fixes
5. **Prevent Recurrence**: Implement safeguards
