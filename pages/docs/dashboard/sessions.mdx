# Managing Sessions

Learn how to view and manage sessions in the dashboard.

## Accessing Sessions

Navigate to **Sessions** in the dashboard sidebar to view all user sessions.

## Session List

The sessions page shows:

- **Session ID**: Unique identifier
- **User**: User identifier
- **Start Time**: When session began
- **Last Activity**: Last interaction time
- **Duration**: Total session duration
- **Messages**: Number of messages
- **Status**: Active or ended

## Session Details

Click on a session to see detailed information:

### Session Information
- Session ID
- User ID
- Start and end times
- Duration
- Status

### Conversation History
- All messages in the session
- Message order and timing
- Query and response pairs
- Context flow

### Analytics
- Total messages
- Average latency
- Total cost
- Error rate
- User satisfaction (if tracked)

## Filtering Sessions

Filter sessions by:

- **Date Range**: Select time period
- **User**: Filter by user ID
- **Status**: Active or ended
- **Duration**: Filter by length
- **Search**: Search by session ID

## Session Management

### Viewing Active Sessions

View currently active sessions:

1. Navigate to **Sessions**
2. Filter by "Status: Active"
3. See all ongoing sessions
4. Monitor real-time activity

### Ending Sessions

Sessions end automatically after inactivity, but you can manually end them through the API.

### Session Analytics

Analyze sessions to:

- Understand user behavior
- Identify patterns
- Improve user experience
- Debug issues
- Track engagement

## Best Practices

1. **Monitor Active Sessions**: Track ongoing interactions
2. **Analyze Patterns**: Look for common conversation flows
3. **Identify Issues**: Find problematic sessions
4. **Track Engagement**: Measure user interaction
5. **Optimize Experience**: Improve based on data
